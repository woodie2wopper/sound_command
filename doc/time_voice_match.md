# コマンドの説明

- time_voice_match.py
  - 音声データと時間データをマッチングする
  - 音声データはwavファイル
  - 時間データはRavenProのannotation TableData

- 入力オプション
  - --help, -h: ヘルプを表示する
  - --debug, -d: デバッグ情報を出力する
  - --column_name_time, -t: 時間データの列名を指定する(デフォルトは"Begin Clock Time")
  - --column_name_voice, -v: 音声データの列名を指定する(デフォルトは"type")
  - --allowed_time_diff, -t: 時間の許容誤差を秒単位で指定する(デフォルトは10秒)
  - --show_diff, -s: 時差"allow_time_diff"秒以内の不一致のデータを最初のファイルのみある場合，２番目のファイルにある場合に"-","+"で表示し，一致した場合には"="で表示する（デフォルトはfalse）
  - --separator, -s: データの区切り文字を指定する(デフォルトは"\t")

- 出力：
  - show_diff==trueなら標準出力に不一致とデータを各行表示し，最終行に一致数と不一致数を表示
  - show_diff==falseなら標準出力に一致数と不一致数のみを表示

- エラー：
  - ファイルが２つでない場合
  - 時間データの列名が存在しない場合
  - 音声データの列名が存在しない場合
  - 時間データの列名が存在しない場合

- アルゴリズム：
1. **データの読み込み**:
- 2つのファイルを読み込み、それぞれのデータをデータフレームやリストとして保持します。
2. **データの前処理**:
  - 時刻のフォーマットを統一し、必要に応じて秒単位や分単位に丸めます。
  - 声の種類の表記揺れを統一します（例: 大文字小文字の違いを無視するなど）。
3. **一致度の計算**:
  - 各ファイルのデータを時刻と声の種類のペアとして比較します。
  - 完全一致するペアの数をカウントします。
4. **不一致度の計算**:
  - 一方のファイルに存在し、もう一方のファイルに存在しないペアの数をカウントします。
  - 逆も同様にカウントします。
5. **一致度と不一致度の指標化**:
  - 一致度を全体のデータ数に対する割合として計算します。
  - 不一致度も同様に割合として計算します。
6. **考慮すべき点**:
  - 時刻の精度: 時刻の一致をどの程度厳密にするか（例: 秒単位、分単位）。→ time_diff_secを設定
  - 声の種類の分類: 声の種類が異なる場合の扱い（例: 類似の声を同一視するかどうか）。→ 不一致とみなす
  - データの欠損: 片方のファイルにしかないデータの扱い。→ 不一致とみなす
